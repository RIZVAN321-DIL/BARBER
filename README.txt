✂️ Бот для парикмахера — ФИНАЛЬНАЯ ВЕРСИЯ (исправленная)

✅ ФУНКЦИИ:
- Клиент: запись через календарь или "Ближайшее время"
- Имя и телефон — опционально (кнопка "Пропустить")
- Отмена записи клиентом за 2+ часа
- Админ: все записи, быстрая запись по звонку, отмена по ID, статистика
- Чёрный список: /ban и /unban (только админ)
- Напоминания: за 24 часа и за 2 часа (без дублей)
- График: 8:00–17:00 (слоты по 1 часу), настраивается в config.py
- Кнопки «Назад» на всех шагах
- Защита от двойной записи + атомарные транзакции в БД
- Логирование и обработка ошибок

⚠️ ПЕРЕД ЗАПУСКОМ:
1. Установите переменные окружения на Bothost:
   BOT_TOKEN=ваш_токен
   ADMIN_ID=ваш_цифровой_ID_админа
   MASTER_PHONE="+7 (900) 123-45-67"
   TZ=Europe/Moscow  ← КРИТИЧНО для правильного времени!

2. Установите зависимости:
   pip install -r requirements.txt

3. Проверьте синтаксис:
   python -m py_compile *.py

4. Запустите:
   python main.py

📁 СТРУКТУРА:
- config.py — настройки и график
- database.py — работа с SQLite
- keyboards.py — все клавиатуры
- main.py — точка входа (запускать его)
- requirements.txt — зависимости
- barber.db — создастся автоматически

🔧 ТЕСТЫ ПЕРЕД ДЕПЛОЕМ:
□ Запись клиента (ручной выбор + авто)
□ Пропуск имени/телефона
□ Отмена записи клиентом
□ Админ: просмотр всех записей
□ Админ: быстрая запись по звонку
□ Админ: отмена по ID
□ Команды /ban и /unban
□ Напоминания (можно протестировать, изменив время в БД)

🆘 Если что-то пошло не так:
- Проверьте логи (выводятся в консоль)
- Убедитесь, что BOT_TOKEN верный
- Проверьте, что ADMIN_ID — число, а не строка
- Убедитесь, что часовой пояс сервера совпадает с вашим

Удачи! 💈✨
